## 1. Conceptos Fundamentales

### 1.1 Fases de Periodizaci√≥n (Im√°genes 1-5)

Motor V3 implementa un modelo de periodizaci√≥n ondulante basado en tres fases c√≠clicas:

#### Fase 1: Acumulaci√≥n (3-6 semanas)

**Objetivo**: M√°xima hipertrofia mediante volumen progresivo

**Caracter√≠sticas**:
- **Volumen**: Incremento semanal de +1-4 sets por m√∫sculo
- **RIR**: Constante (2-3 RIR en mayor√≠a de series)
- **Intensidad (%1RM)**: Moderada (60-80%)
- **Fatiga**: Acumulativa pero manejable

**Progresi√≥n tipo**:
```
Semana 1: 12 sets/m√∫sculo @ 2-3 RIR
Semana 2: 14 sets/m√∫sculo @ 2-3 RIR
Semana 3: 16 sets/m√∫sculo @ 2-3 RIR
Semana 4: 18 sets/m√∫sculo @ 2-3 RIR (cerca MRV)
```

**Mecanismo**:
- Mayor volumen total ‚Üí Mayor tiempo bajo tensi√≥n
- Da√±o muscular controlado ‚Üí Respuesta hipertr√≥fica
- Fatiga metab√≥lica ‚Üí Mejora capacidad glucol√≠tica

**Evidencia**:
- Schoenfeld et al. (2019): Volumen progresivo superior a volumen est√°tico
- Helms et al. (2018): Acumulaci√≥n de volumen necesaria para adaptaci√≥n sostenida

**Se√±ales de fin de fase**:
- ‚úÖ Rendimiento estable o mejorando
- ‚ùå Ca√≠da de rendimiento >10% en ejercicios principales
- ‚ùå Fatiga sist√©mica excesiva (DOMS prolongado >4 d√≠as)
- ‚ùå Alcance de MRV individual

#### Fase 2: Mantenimiento (1-2 semanas)

**Objetivo**: Pico de intensidad y consolidaci√≥n de ganancias

**Caracter√≠sticas**:
- **Volumen**: Constante o ligera reducci√≥n (-10-20%)
- **RIR**: Descendente (3‚Üí2‚Üí1‚Üí0 RIR)
- **Intensidad (%1RM)**: Alta (80-90%)
- **Fatiga**: Disminuci√≥n gradual

**Progresi√≥n tipo**:
```
Semana 5: 16 sets/m√∫sculo @ 1-2 RIR
Semana 6: 14 sets/m√∫sculo @ 0-1 RIR (series al fallo)
```

**Mecanismo**:
- Reducci√≥n volumen ‚Üí Menor fatiga acumulada
- Mayor intensidad ‚Üí Pico neural y fuerza
- Series al fallo ‚Üí Reclutamiento fibras tipo II m√°ximo

**Evidencia**:
- Gonz√°lez-Badillo et al. (2016): Intensificaci√≥n post-acumulaci√≥n mejora pico de fuerza
- Pareja-Blanco et al. (2017): Reducci√≥n volumen + aumento intensidad = consolidaci√≥n

**Aplicaci√≥n**:
- Preparaci√≥n para test de fuerza (ej: probar nuevos 1RM)
- Finalizaci√≥n de mesociclo antes de deload
- Transici√≥n entre bloques de entrenamiento

#### Fase 3: Regeneraci√≥n/Deload (1 semana cada 2-4 semanas acumulaci√≥n)

**Objetivo**: Recuperaci√≥n sist√©mica completa y resensibilizaci√≥n

**Caracter√≠sticas**:
- **Volumen**: Reducci√≥n 40-60% del volumen de acumulaci√≥n
- **RIR**: Alto (3-4 RIR, muy lejos del fallo)
- **Intensidad (%1RM)**: Moderada (65-75%)
- **Fatiga**: Disipaci√≥n completa

**Estrategias de deload**:

1. **Reducci√≥n de volumen** (recomendada):
   - Mantener ejercicios e intensidad
   - Reducir sets: 4 sets ‚Üí 2 sets por ejercicio

2. **Reducci√≥n de intensidad**:
   - Mantener ejercicios y sets
   - Reducir carga: -20-30% peso habitual

3. **Reducci√≥n de frecuencia**:
   - Reducir d√≠as de entrenamiento: 4 d√≠as ‚Üí 2 d√≠as

**Progresi√≥n tipo**:
```

```

**Mecanismo**:
- Disipaci√≥n de fatiga perif√©rica (da√±o muscular, depleci√≥n gluc√≥geno)
- Recuperaci√≥n neural (SNC)
- Reparaci√≥n de microtraumas articulares y tendinosos
- Resensibilizaci√≥n a est√≠mulo (evita acomodaci√≥n)

**Evidencia**:
- Pritchard et al. (2015): Deload cada 4-6 semanas mejora progreso largo plazo
- Zourdos et al. (2016): Reducci√≥n volumen 40-60% √≥ptima para recuperaci√≥n

**Se√±ales de necesidad de deload**:
- ‚ùå Rendimiento estancado o decreciente >2 semanas
- ‚ùå Fatiga acumulada (sensaci√≥n de cansancio cr√≥nico)
- ‚ùå Dolor articular persistente
- ‚ùå Alteraciones del sue√±o o apetito
- ‚ùå P√©rdida de motivaci√≥n

**Frecuencia de deload**:
- **Principiantes**: Cada 4-6 semanas
- **Intermedios**: Cada 3-4 semanas
- **Avanzados**: Cada 2-3 semanas (mayor volumen = mayor fatiga)

### 1.2 Variaci√≥n Vertical de Ejercicios (Im√°genes 6-9)

La variaci√≥n vertical es el cambio estrat√©gico de ejercicios para mantener adaptaci√≥n continua sin sacrificar especificidad.

#### Modelo de Inflexi√≥n

**Fases de respuesta a ejercicio nuevo**:

1. **Novedad** (Semanas 1-2):
   - Alta estimulaci√≥n neural (aprendizaje motor)
   - Reclutamiento err√°tico de fibras
   - Adaptaci√≥n r√°pida inicial

2. **Adaptaci√≥n** (Semanas 3-8):
   - Zona √≥ptima de est√≠mulo
   - Eficiencia neural estable
   - Progresi√≥n lineal de carga/volumen

3. **Inflexi√≥n** (Semanas 9+):
   - Rendimientos decrecientes
   - Acomodaci√≥n neuromuscular
   - Necesidad de cambio

**Ventana √≥ptima**: Mantener ejercicio 6-12 semanas

**Evidencia**:
- Fonseca et al. (2014): Variaci√≥n cada 8 semanas superior a variaci√≥n diaria
- Baz-Valle et al. (2019): Especificidad > variaci√≥n para hipertrofia (variar solo cuando necesario)

#### Tipos de Variaci√≥n

**Variaci√≥n completa** (cambio de movimiento):
```
Bench Press ‚Üí Incline DB Press
(Horizontal ‚Üí Angular + unilateral)
```

**Variaci√≥n parcial** (cambio de implemento/rango):
```
Barbell Row ‚Üí DB Row
(Bilateral ‚Üí Unilateral, mismo patr√≥n)

Squat ‚Üí Front Squat
(√ânfasis posterior ‚Üí anterior, mismo patr√≥n)
```

**Variaci√≥n m√≠nima** (cambio de grip/stance):
```
Barbell Curl ‚Üí EZ-Bar Curl
(Stress mu√±eca reducido, mismo movimiento)

Conventional Deadlift ‚Üí Sumo Deadlift
(√ânfasis isquios ‚Üí gl√∫teos/aductores)
```

#### Directrices de Variaci√≥n

1. **Ejercicios principales** (compuestos multi-articulares):
   - Mantener 8-12 semanas
   - Cambiar solo cuando progreso estancado >2 semanas

2. **Ejercicios accesorios** (aislamiento/compuestos secundarios):
   - Mantener 4-8 semanas
   - Mayor libertad de variaci√≥n

3. **Usuarios principiantes**:
   - Minimizar variaci√≥n (mantener 12-16 semanas)
   - Prioridad: Dominio t√©cnico

4. **Usuarios avanzados**:
   - Variaci√≥n estrat√©gica cada 6-8 semanas
   - Mantener patr√≥n de movimiento, variar implemento

**Anti-patr√≥n**: Variaci√≥n excesiva
```
‚ùå Cambiar ejercicios cada semana (no permite adaptaci√≥n)
‚ùå Cambiar sin raz√≥n (ej: por aburrimiento vs. estancamiento)
```

enum TrainingPhase {
  accumulation,   // Volumen ‚Üë
  intensification, // RIR ‚Üì
  deload,         // Recuperaci√≥n
}

class PeriodizationEngine {
  /// Determina fase actual del mesociclo
  static TrainingPhase determineCurrentPhase({
    required int weekInMesocycle,
    required PerformanceMetrics metrics,
  }) {
    // Estructura t√≠pica: 4 semanas acumulaci√≥n + 1 intensificaci√≥n + 1 deload
    final weeksInAccumulation = 4;
    final weeksInIntensification = 1;

    if (metrics.requiresImmediateDeload) {
      return TrainingPhase.deload; // Override por fatiga
    }

    if (weekInMesocycle <= weeksInAccumulation) {
      return TrainingPhase.accumulation;
    } else if (weekInMesocycle <= weeksInAccumulation + weeksInIntensification) {
      return TrainingPhase.intensification;
    } else {
      return TrainingPhase.deload;
    }
  }

  /// Calcula volumen objetivo para la semana
  static int calculateWeeklyVolume({
    required TrainingPhase phase,
    required int baselineVolume, // MAV del usuario
    required int weekInPhase,
  }) {
    switch (phase) {
      case TrainingPhase.accumulation:
        // Progresi√≥n lineal: +2 sets/semana
        return baselineVolume + (weekInPhase * 2);

      case TrainingPhase.intensification:
        // Mantener volumen alto o reducir 10%
        return (baselineVolume * 0.9).round();

      case TrainingPhase.deload:
        // Reducir 50% del volumen
        return (baselineVolume * 0.5).round();
    }
  }

  /// Calcula RIR objetivo para la semana
  static double calculateTargetRIR({
    required TrainingPhase phase,
    required int weekInPhase,
  }) {
    switch (phase) {
      case TrainingPhase.accumulation:
        return 2.5; // 2-3 RIR constante

      case TrainingPhase.intensification:
        // RIR descendente: 2 ‚Üí 1 ‚Üí 0
        return 2.0 - weekInPhase; // Semana 1: 2 RIR, Semana 2: 1 RIR

      case TrainingPhase.deload:
        return 4.0; // 3-4 RIR (muy conservador)
    }
  }
}

/// M√©tricas de rendimiento del usuario
class PerformanceMetrics {

  final double sleepQuality;    // 1-10
  final double energyLevel;     // 1-10
  final int jointPain;          // 0-10
  final int domsIntensity;      // 0-10

  const PerformanceMetrics({
    required this.loadProgression,
    required this.sleepQuality,
    required this.energyLevel,
    required this.jointPain,
    required this.domsIntensity,
  });

  /// Determina si usuario requiere deload inmediato
  bool get requiresImmediateDeload {
    return (sleepQuality < 6 && energyLevel < 5) ||
        jointPain > 6 ||
        loadProgression < -0.10; // Ca√≠da >10% en carga
  }

  /// Score general de recuperaci√≥n (0-1)
  double get recoveryScore {
    final sleepScore = sleepQuality / 10;
    final energyScore = energyLevel / 10;
    final painScore = 1 - (jointPain / 10);
    final domsScore = 1 - (domsIntensity / 10);

    return (sleepScore + energyScore + painScore + domsScore) / 4;
  }
}

/// lib/features/training/domain/engines/reactive_monitoring_engine.dart

class ReactiveMonitoringEngine {
  /// Ajusta programaci√≥n basado en rendimiento real
  static WorkoutAdjustment analyzeAndAdjust({
    required WorkoutSession completedSession,
    required WorkoutSession plannedNextSession,
    required PerformanceMetrics metrics,
  }) {
    // Regla 1: Deload inmediato por fatiga
    if (metrics.requiresImmediateDeload) {
      return WorkoutAdjustment(
        type: AdjustmentType.forceDeload,
        reason: 'Marcadores de fatiga elevados',
        volumeChange: -0.5, // -50%
        rirChange: +2.0,    // +2 RIR
      );
    }

    // Regla 2: Ajuste por RIR discordante
    final rirDiscordance = _analyzeRIRDiscordance(completedSession);
    if (rirDiscordance.needsAdjustment) {
      return WorkoutAdjustment(
        type: AdjustmentType.loadAdjustment,
        reason: 'RIR reportado ${rirDiscordance.direction} objetivo',
        loadChange: rirDiscordance.suggestedLoadChange,
      );
    }

    // Regla 3: Ajuste por ca√≠da de rendimiento
    final performanceDrop = _analyzePerformanceDrop(
      completedSession,
      metrics.loadProgression,
    );
    if (performanceDrop > 0.10) {
      return WorkoutAdjustment(
        type: AdjustmentType.volumeReduction,
        reason: 'Ca√≠da de rendimiento ${(performanceDrop * 100).toStringAsFixed(0)}%',
        volumeChange: -0.2, // -20% volumen
      );
    }

    // Regla 4: Progreso sostenido = acelerar progresi√≥n
    if (metrics.loadProgression > 0.05 && metrics.recoveryScore > 0.7) {
      return WorkoutAdjustment(
        type: AdjustmentType.accelerateProgression,
        reason: 'Progreso sostenido y buena recuperaci√≥n',
        volumeChange: +0.1, // +10% volumen
      );
    }

    // No se necesita ajuste
    return WorkoutAdjustment.noChange();
  }

  /// Analiza discordancia entre RIR objetivo y real
  static RIRDiscordanceAnalysis _analyzeRIRDiscordance(
    WorkoutSession session,
  ) {
    final sets = session.completedSets;
    int setsAboveTarget = 0;
    int setsBelowTarget = 0;

    for (final set in sets) {
      final difference = set.actualRIR - set.targetRIR;
      if (difference > 1.5) {
        setsAboveTarget++; // RIR muy alto = carga muy baja
      } else if (difference < -1.5) {
        setsBelowTarget++; // RIR muy bajo = carga muy alta
      }
    }

    if (setsAboveTarget >= 3) {
      return RIRDiscordanceAnalysis(
        needsAdjustment: true,
        direction: 'superior a',
        suggestedLoadChange: +0.05, // Incrementar 5%
      );
    } else if (setsBelowTarget >= 3) {
      return RIRDiscordanceAnalysis(
        needsAdjustment: true,
        direction: 'inferior a',
        suggestedLoadChange: -0.10, // Reducir 10%
      );
    }

    return RIRDiscordanceAnalysis.noAdjustment();
  }

  /// Analiza ca√≠da de rendimiento
  static double _analyzePerformanceDrop(
    WorkoutSession session,
    double loadProgression,
  ) {
    // Ca√≠da de rendimiento = loadProgression negativo
    return loadProgression < 0 ? loadProgression.abs() : 0.0;
  }
}

class WorkoutAdjustment {
  final AdjustmentType type;
  final String reason;
  final double? volumeChange;  // Multiplicador (ej: -0.2 = -20%)
  final double? loadChange;    // Multiplicador (ej: +0.05 = +5%)
  final double? rirChange;     // Delta (ej: +2 = +2 RIR)

  const WorkoutAdjustment({
    required this.type,
    required this.reason,
    this.volumeChange,
    this.loadChange,
    this.rirChange,
  });

  factory WorkoutAdjustment.noChange() {
    return const WorkoutAdjustment(
      type: AdjustmentType.none,
      reason: 'Progreso seg√∫n lo planeado',
    );
  }
}

enum AdjustmentType {
  none,
  forceDeload,
  loadAdjustment,
  volumeReduction,
  accelerateProgression,
}

class RIRDiscordanceAnalysis {
  final bool needsAdjustment;
  final String direction;
  final double suggestedLoadChange;

  const RIRDiscordanceAnalysis({
    required this.needsAdjustment,
    required this.direction,
    required this.suggestedLoadChange,
  });

  factory RIRDiscordanceAnalysis.noAdjustment() {
    return const RIRDiscordanceAnalysis(
      needsAdjustment: false,
      direction: '',
      suggestedLoadChange: 0.0,
    );
  }
}
